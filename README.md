# RescueAI — Predictive Emergency Resource Allocation

RescueAI is a lightweight decision-support interface designed to help emergency responders anticipate risk concentration across urban zones and prepare resources *before* incidents escalate.

This prototype currently focuses on Bengaluru as a pilot deployment environment.

The system visualizes location-based risk levels on an interactive map and provides short AI-assisted explanations to support emergency placement decisions.

Built as part of the Google Solution Challenge 2026.

---

## Why this project exists

Emergency response systems are typically reactive.

Vehicles are dispatched *after* incidents occur rather than positioned ahead of time based on risk signals like weather conditions, congestion timing, and known accident-prone corridors.

RescueAI explores whether simple environmental indicators combined with interpretable AI explanations can support earlier operational decisions.

The goal is not prediction perfection.

The goal is **actionable situational awareness**.

---

## What the prototype currently does

The interface allows users to:

• view selected Bengaluru zones on a live map  
• simulate risk scenarios without backend connectivity  
• request real-time risk scores from a prediction API  
• read short deployment-oriented explanations generated by Gemini  

Each zone is rendered visually using color-coded risk indicators:

Green → low  
Orange → moderate  
Red → high  

Selecting a zone reveals its current risk interpretation and recommended response posture.

---

## Why Bengaluru

Bengaluru presents a strong pilot environment because:

high traffic variability  
weather-influenced congestion patterns  
dense junction clusters  
rapid emergency demand shifts across zones  

The same interface can later be extended to additional cities without redesigning the frontend architecture.

---

## What makes this different from a static map demo

Most student projects visualize locations.

This system attempts to visualize **decision context**.

Risk is not shown as a number alone.

Each zone includes a short explanation describing *why* the system suggests a deployment posture.

This keeps the interface interpretable for responders rather than opaque.
